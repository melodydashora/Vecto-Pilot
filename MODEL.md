## Perplexity AI

As of December 2025, Perplexity has unified its API offering under the **Sonar** brand, deprecating previous `llama-3-sonar` and `pplx` model names. The absolute latest production models are **Sonar Pro** and **Sonar**.

### 1. Production Models (December 2025)

These are the exact strings to use in the `model` parameter.

| Model ID | Context Window | Description |
| :--- | :--- | :--- |
| **`sonar-pro`** | **200,000 tokens** | **Flagship Model.** High-intelligence, multi-step reasoning, huge context. Optimized for complex research and "Pro Search" capabilities. |
| **`sonar`** | **127,000 tokens** | **Standard Model.** Lightweight, lower latency, cost-effective. Best for quick, factual lookups. |
| **`sonar-reasoning`** | 127,000 tokens | **Reasoning Model.** Specializes in chain-of-thought (CoT) processing combined with real-time search. |

> **Note:** Previous model IDs like `llama-3.1-sonar-huge-128k-online` are deprecated. Update legacy code to use the short names above.

#### **Endpoint URL**
**POST** `https://api.perplexity.ai/chat/completions`

#### **Authorization**
Header: `Authorization: Bearer <YOUR_API_KEY>`

#### **Key Parameters**

The API is OpenAI-compatible but includes critical proprietary parameters for controlling search behavior.

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `model` | string | **Yes** | Use `sonar-pro` or `sonar`. |

| `search_recency_filter`| string | No | **Important.** Restrict search time window. Values: `"month"`, `"week"`, `"day"`, `"hour"`. |

## SerpAPI

The following technical report details the production status of SerpAPI as of December 2025.

### **Crucial Technical Clarification**
**SerpAPI is not an LLM provider.** It is a real-time API that scrapes and parses search engine results.
*   **No "Context Window":** Replaced by **SERP Parsing** (HTML to structured JSON).
*   **No "Token Pricing":** Replaced by **Per-Search Pricing**.
*   **No "Model IDs":** Replaced by **Engine Parameters** (`engine=...`).

The "Newest Models" in this context refer to SerpAPI's support for scraping **Google's AI Features** (AI Overviews and AI Mode).

### **4. Critical Integration Parameters**

The parameter schema changed significantly between Gemini 2.5 and Gemini 3 regarding "Thinking" (Reasoning) capabilities.

#### **For Gemini 3 Pro (`gemini-3-pro-preview`)**
Use **`thinking_level`**. Do *not* use `thinking_budget`.
*   **Parameter**: `thinking_level`
*   **Type**: String
*   **Allowed Values**:
    *   `"high"` (Default): Maximizes reasoning depth. Higher latency, higher cost.
    *   `"low"`: Minimizes latency/cost. Good for simpler instructions.
    *   *(Note: "medium" is planned but not active as of Dec 10, 2025)*.
*   **New Vision Parameter**: `media_resolution`
    *   **Values**: `"low"`, `"medium"`, `"high"`
    *   **Usage**: Controls token usage/fidelity for image inputs.

**JSON Payload Example (Gemini 3):**
```json
{
  "contents": [{ "parts": [{ "text": "Explain quantum entanglement." }] }],
  "generationConfig": {
    "thinking_config": {
      "thinking_level": "high" 
    },
    "media_resolution": "medium"
  }
}
```

#### **For Gemini 2.5 Family (`gemini-2.5-pro`, `gemini-2.5-flash`)**
Use **`thinking_budget`**.
*   **Parameter**: `thinking_budget`
*   **Type**: Integer (Token count)
*   **Values**:
    *   `0`: Disables thinking (Flash/Flash-Lite only).
    *   `-1`: Dynamic thinking (Model decides, capped at ~8k tokens).
    *   `512` - `32768`: Manual token limit for reasoning traces.

**JSON Payload Example (Gemini 2.5):**
```json
{
  "contents": [{ "parts": [{ "text": "Solve this logic puzzle..." }] }],
  "generationConfig": {
    "thinking_config": {
      "thinking_budget": 2048
    }
  }
}
```

### **5. Special Capabilities Summary**
*   **Reasoning (Thinking)**: Available on all listed models. Gemini 3 uses abstract "levels"; Gemini 2.5 uses precise "token budgets".
*   **Thought Signatures**: Gemini 3 returns encrypted "thought signatures". For multi-turn conversations involving reasoning, you **must** pass these signatures back in the history to maintain context state, otherwise performance degrades.
*   **Vision**: Gemini 3 Pro allows granular control over image token costs via `media_resolution`.
*   **JSON Mode**: Fully supported on all models via `response_mime_type: "application/json"`.

### **3. Exact API Implementation Details**

#### **Request Body Structure**
The following JSON structure applies to both models, with the specific "Thinking" block required to enable reasoning.

```json
{
  "model": "claude-sonnet-4-5-20250929",
  "max_tokens": 8192,
  "messages": [
    {"role": "user", "content": "Explain the quantum hall effect."}
  ],
  "thinking": {
    "type": "enabled",
    "budget_tokens": 4096
  }
}
```

#### **Supported Parameters**
| Parameter | Description | Notes |
| :--- | :--- | :--- |
| `model` | **Required**. The model ID string. | Use `claude-sonnet-4-5` or `claude-3-7-sonnet-latest`. |
| `messages` | **Required**. Array of message objects. | `[{"role": "user", "content": "..."}]` |
| `max_tokens` | **Required**. Hard limit on output tokens. | Must be >= `budget_tokens`. |
| `thinking` | **Optional**. Enables reasoning. | object: `{"type": "enabled", "budget_tokens": int}` |
| `temperature` | Controls randomness (0.0 - 1.0). | **Exclusive** with `top_p` in Sonnet 4.5. |
| `top_p` | Nucleus sampling (0.0 - 1.0). | **Exclusive** with `temperature` in Sonnet 4.5. |
| `top_k` | Limits vocabulary to top K options. | Supported on both. |
| `tools` | Array of tool definitions. | JSON Schema format. |
| `tool_choice` | Forces/disables tool use. | `auto`, `any`, or `tool`. |
| `stream` | Boolean. | Set `true` for streaming responses. |

#### **"Extended Thinking" Configuration**
To use the reasoning/thinking capabilities found in Sonnet 3.7 and 4.5:
1.  Add the `thinking` parameter to your request body.
2.  `type`: Must be set to `"enabled"`.
3.  `budget_tokens`: An integer (minimum **1024**). This counts towards your `max_tokens` limit and is billed as **output** tokens.

**Example Python Snippet (Sonnet 4.5):**
```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=20000,
    temperature=1.0, # Do NOT set top_p if setting temperature
    thinking={
        "type": "enabled",
        "budget_tokens": 16000
    },
    messages=[
        {"role": "user", "content": "Solve this complex logic puzzle..."}
    ]
)
print(response.content)
```

## OpenAI

Based on verifiable research for **December 2025**, here are the absolute latest production models and API integration details for OpenAI.

### **Executive Summary: December 2025 Model Landscape**
As of late 2025, OpenAI has shifted toward the **`v1/responses`** API endpoint for its newest frontier models (`GPT-5.1`, `o3`, `o4-mini`). While the legacy `chat/completions` endpoint remains supported for older models, the **Responses API** is required or highly recommended for the new "agentic" and "reasoning" capabilities (Thinking/Instant modes).

### **1. Top Production Models (Verified Dec 2025)**

| :--- | :--- | :--- | :--- | :--- |

> **Note:** Pricing reflects the "standard" tier. `o3-pro` and specialized coding variants (`gpt-5.1-codex-max`) exist at higher price points.
> *   **Cached Input Price:** Typically ~10-25% of the standard input price (e.g., $0.125/1M for GPT-5.1).

---

### **2. Integration Details: The New Standard**

The newest models utilize the **Responses API** (`/v1/responses`) to support agentic loops, multiple tool calls, and advanced reasoning controls.

#### **A. Exact Endpoint URL**
```http
POST https://api.openai.com/v1/responses
```
*(Legacy `v1/chat/completions` may work for basic text but does not support full agentic/reasoning features of GPT-5.1/o3)*

#### **B. Required Payload Structure (JSON)**
The `v1/responses` endpoint uses `input` (list of items) instead of `messages`.

```json
{
  "model": "gpt-5.1",
  "reasoning": {
    "effort": "medium" 
  },
  "input": [
    {
      "role": "user",
      "content": "Analyze this P&L statement for Q4 trends."
    }
  ],
  "tools": [
    {
      "type": "web_search"
    }
  ],
  "max_completion_tokens": 16384
}
```

#### **C. Key Parameters**

1.  **`reasoning_effort`** (Critical for GPT-5.1 & o3):
    *   **Supported Values:** `none`, `low`, `medium`, `high`.
    *   **Default:** `none` for **GPT-5.1** (Instant mode).
    *   **Default:** `medium` for **o3 / o4-mini**.
    *   *Usage:* Set to `high` for complex coding/math; `none` for lowest latency chat.

2.  **`max_completion_tokens`**:
    *   **Mandatory** replacement for `max_tokens` when using reasoning models.
    *   Includes both visible output tokens and invisible "reasoning" (CoT) tokens.

---

### **3. Detailed Model Breakdown**

#### **GPT-5.1 (Flagship)**
*   **Best For:** Agentic workflows, complex instructions, general-purpose "thinking" or "instant" tasks.
*   **Special Capabilities:**
    *   **Adaptive Reasoning:** Can switch between "Instant" (Chat) and "Thinking" modes via the `reasoning.effort` parameter.
    *   **Agentic Workflows:** Native support for multi-step tool use (Web Search, File Search, Code Interpreter) in a single API call loop.
    *   **Steerability:** Significantly improved adherence to system prompts and tone settings.
    *   **Variants:** `gpt-5.1-codex-max` (specialized for long-context coding).

#### **o3 (SOTA Reasoning)**
*   **Best For:** STEM, Deep Research, Complex Visual Analysis.
*   **Special Capabilities:**
    *   **Visual Reasoning:** Can "reason" about images/charts with chain-of-thought (unlike previous text-only reasoning).
    *   **Deep Research:** Capable of autonomous "deep research" loops (available via specific `o3-deep-research` variants or tool configurations).
    *   **Performance:** Replaces `o1` as the state-of-the-art for Math/Code benchmarks.

#### **o4-mini (Cost/Speed Leader)**
*   **Best For:** High-volume reasoning tasks, fast code generation, real-time analysis.
*   **Special Capabilities:**
    *   **Multimodal:** Processes text and images with reasoning.
    *   **Efficiency:** Designed to compete with "Flash" models but with "reasoning" capabilities enabled.
    *   **Pricing:** Extremely aggressive pricing ($1.10/1M input) makes it viable for high-throughput applications.

### **4. Migration Quick-List**
*   **Update SDKs:** Ensure you are using `openai>=2.0.0` (Python/Node) to access `v1/responses`.
*   **Replace `max_tokens`:** Global find/replace to `max_completion_tokens`.
*   **Set `reasoning_effort`:** Explicitly set this to `medium` or `high` if you are migrating from `o1` to ensure consistent depth, as GPT-5.1 defaults to `none`.
