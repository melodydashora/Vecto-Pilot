The "Newest Models" in this context are **Google's Gemini-powered AI Overviews** (formerly SGE), which SerpAPI exposes via specific engine parameters.

### 3. Critical Integration Parameters

There is a **breaking change** in how reasoning is controlled between Gemini 3 and Gemini 2.5.

#### **A. Gemini 3 Pro Preview (`gemini-3-pro-preview`)**
Uses `thinking_level` (Enum) instead of a raw token budget.
*   **Parameter:** `thinking_level`
*   **Values:** `"LOW"`, `"HIGH"` (Default: Dynamic/High)
*   **Behavior:** You cannot disable thinking completely. `LOW` optimizes for latency/throughput; `HIGH` optimizes for depth.
*   **New Parameter:** `media_resolution` (Values: `"low"`, `"medium"`, `"high"`) controls vision token usage.

**JSON Payload Example (Gemini 3):**
```json
{
  "contents": [{ "parts": [{ "text": "Analyze this architectural diagram." }] }],
  "generationConfig": {
    "thinking_config": { "thinking_level": "HIGH" },
    "media_resolution": "medium"
  }
}
```

#### **B. Gemini 2.5 Family (`gemini-2.5-pro`, `gemini-2.5-flash`)**
Uses `thinking_budget` (Integer) to define the token limit for reasoning.
*   **Parameter:** `thinking_budget`
*   **Values:** Integer (e.g., `1024`, `8192`). Set to `0` to **disable** thinking on Flash/Flash-Lite.
*   **Constraint:** You cannot disable thinking on `gemini-2.5-pro`.

**JSON Payload Example (Gemini 2.5):**
```json

  "contents": [{ "parts": [{ "text": "Explain quantum entanglement." }] }],

    "thinking_config": { "thinking_budget": 4096 }
  }
}
```

### **5. Critical Parameter Changes**
1.  **`reasoning_effort`**:
    *   **GPT-5.1:** Supports `none` (critical for speed/cost savings).
    *   **o3 / o4-mini:** Only `low`, `medium`, `high`.
2.  **`max_completion_tokens`**: Use this instead of `max_tokens` for all reasoning models (o3, o4-mini, GPT-5.1) to guarantee space for both "thinking" tokens and visible output.
3.  **`role: "developer"`**: Replaces `"system"` for guiding model behavior in newer models.
