## SerpAPI

**CONTEXT: TECHNICAL CLARIFICATION**
SerpAPI is **not an LLM provider** (like OpenAI or Anthropic). It is a **real-time search engine scraping API** that converts search results (Google, Bing, etc.) into structured JSON.

It does not have "context windows" or "token-based pricing." It charges per **successful search**.
However, it **does** have a new engine specifically designed to scrape Google's **AI Overviews** (Generative AI results), which is the closest equivalent to your request.

### **1. NEWEST PRODUCTION ENGINES (Dec 2025)**
The most relevant new engine for AI researchers is the **Google AI Overview API**, which extracts the generative AI text blocks from Google Search.

| Engine Name | Description | Verified Status |
| :--- | :--- | :--- |
| **Google AI Overview** | Scrapes the "AI Overview" (formerly SGE) block from Google Search. | **Production** |
| **Google Events** | Scrapes rich event data (dates, venues, tickets) from Google. | **Production** |
| **Google Light** | A stripped-down, faster version of Google Search (good for LLM contexts). | **Production** |

---

### **2. EXACT API INTEGRATION DETAILS**

#### **A. Base Endpoint**
*   **Method:** `GET`
*   **Base URL:** `https://serpapi.com/search.json`
*   **SDK (Python):** `pip install google-search-results`

#### **B. Google Events Engine (`google_events`)**
Use this for verifiable, real-time event data.

*   **Model ID (Engine Parameter):** `google_events`
*   **Required Parameters:**
    *   `api_key`: Your secret key.
    *   `engine`: `google_events`
    *   `q`: Search query (e.g., "AI Conferences in San Francisco").
*   **Key Optional Parameters:**
    *   `location`: Full location string (e.g., "Austin, Texas, United States").
    *   `hl`: Language code (e.g., `en`).
    *   `gl`: Country code (e.g., `us`).
    *   `start`: Pagination offset (default `0`).
    *   `htichips`: Filter by date/type (e.g., `date:today`, `event_type:Virtual-Event`).

#### **C. Google AI Overview Engine (`google_ai_overview`)**
Use this to capture the output of Google's Gemini/AI models via Search.

*   **Model ID (Engine Parameter):** `google_ai_overview`
*   **Trigger Logic:** This engine is often used *after* a standard search. If a standard `google` search returns a `page_token` for an AI overview, you pass that token to this engine.
*   **Direct Access:** You can also force it via `engine=google` with specific flags, but `google_ai_overview` is the dedicated endpoint for specific tokens.

### **3. PRICING & LIMITS (December 2025)**

SerpAPI does **NOT** charge per token. It uses a **subscription model** based on the number of searches.

*   **Developer Plan:** $75/month for **5,000 searches**
    *   **Cost per Search:** ~$0.015
*   **Production Plan:** $150/month for **15,000 searches**
    *   **Cost per Search:** ~$0.010
*   **Free Tier:** 250 searches/month (Upgraded in 2025 with "Ludicrous Speed").

**Token Economy Equivalent:**
Since one search returns ~5KB-15KB of JSON, the "cost per 1M tokens" concept doesn't apply directly. However, for context:
*   **Input Cost:** $0 (You pay per search, not per query length).
*   **Output Cost:** Effectively fixed per search results page, regardless of token count.

### **4. SPECIAL CAPABILITIES & "CONTEXT WINDOW"**

While there is no "Context Window," SerpAPI introduced a feature in 2025 to manage payload size for LLMs:

*   **Feature:** **JSON Restrictor**
*   **Parameter:** `json_restrictor`
*   **Function:** Allows you to filter the JSON response *server-side* before it reaches your application.
*   **Use Case:** Critical for LLM integration. You can request *only* the text snippets to save tokens in your LLM's context window.
    *   *Example:* `&json_restrictor=organic_results[0].snippet` (Returns only the snippet of the first result).

#### **New "Thinking" Parameters**
**Crucial Distinction:** The API implementation differs significantly between version 3 and 2.5.

**Option A: Gemini 3 Pro (Uses Levels)**
*   **Parameter:** `thinking_level`
*   **Values:** `LOW`, `HIGH` (Default: `HIGH`)
*   **Behavior:** Controls the depth of reasoning. `LOW` is faster; `HIGH` enables deep, multi-step chain-of-thought.
*   **Code Example (JSON):**
    ```json
    {
      "contents": [{ "parts": [{ "text": "Explain quantum entanglement." }] }],
      "generationConfig": {
        "thinkingConfig": {
          "thinkingLevel": "HIGH" 
        }
      }
    }
    ```

**Option B: Gemini 2.5 Family (Uses Budgets)**
*   **Parameter:** `thinking_budget`
*   **Values:** Integer (Token count). Set to `0` to disable (Flash only).
*   **Behavior:** Sets a hard cap on tokens used for internal reasoning before output generation.
*   **Code Example (JSON):**
    ```json
    {
      "generationConfig": {
        "thinkingConfig": {
          "thinkingBudget": 2048
        }
      }
    }
    ```

### **Integration Details**

#### **API Endpoint**
*   **Primary (New)**: `https://api.openai.com/v1/responses`
*   **Legacy**: `https://api.openai.com/v1/chat/completions`

#### **Required Parameters (JSON Body)**
For `o3`, `o4-mini`, and `gpt-5.1`, you must use **`max_completion_tokens`** instead of `max_tokens`.

```json
{
  "model": "gpt-5.1",
  "messages": [
    { "role": "user", "content": "Explain quantum entanglement." }
  ],
  "reasoning_effort": "medium",
  "max_completion_tokens": 2048,
  "store": true
}
```

#### **Key Parameter Specs**
1.  **`reasoning_effort`**: Controls the density of the chain-of-thought.
    *   **Supported Values**: `low`, `medium`, `high`.
    *   **GPT-5.1 Only**: Supports `none` (bypasses reasoning tokens for "Instant" speed).
    *   **Default**: `medium` for o3/o4-mini; `none` for gpt-5.1 (unless specified).

2.  **`max_completion_tokens`**:
    *   **Mandatory** replacement for `max_tokens` on all reasoning models.
    *   Includes both visible output tokens and invisible reasoning tokens.

3.  **`reasoning_summary`** (Optional):
    *   Values: `auto`, `detailed`.
    *   Allows retrieval of the reasoning trace (if permitted by the specific model policy).

### 1. OpenAI (GPT-5.1)
**Research Context:** Released Nov 2025. Flagship model `gpt-5.1`. The research notes a transition to `v1/responses`, but `v1/chat/completions` remains supported for backward compatibility. These examples use the standard client with the new model ID.

**Installation:**
```bash

```

#### Python SDK
```python
import os
from openai import OpenAI

# Initialize client
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# --- Basic Usage ---
def basic_chat():
    try:
        response = client.chat.completions.create(
            model="gpt-5.1",
            messages=[
                {"role": "user", "content": "Explain quantum entanglement briefly."}
            ]
        )
        print(response.choices[0].message.content)
    except Exception as e:
        print(f"Error: {e}")

# --- Advanced Usage (Streaming + Adaptive Reasoning) ---
def advanced_reasoning():
    try:
        # GPT-5.1 features adaptive reasoning (switching between fast/slow thinking)
        stream = client.chat.completions.create(
            model="gpt-5.1",
            messages=[
                {"role": "system", "content": "You are a senior physicist."},
                {"role": "user", "content": "Solve this complex lattice problem."}
            ],
            temperature=0.7,
            max_tokens=4096,
            stream=True,
            user="user_12345"
        )

        print("Streaming response:")
        for chunk in stream:
            if chunk.choices[0].delta.content is not None:
                print(chunk.choices[0].delta.content, end="")
    except Exception as e:
        print(f"Error: {e}")
```
