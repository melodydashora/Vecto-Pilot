# Session Logs (`docs/memory/sessions/`)

## Purpose

Daily session logs documenting work completed, decisions made, and learnings captured during development sessions.

## Naming Convention

```
YYYY-MM-DD-brief-description.md
```

## Files

| File | Topic |
|------|-------|
| `2025-12-27-router-refactor.md` | Router refactoring session |
| `2025-12-28-map-features.md` | Map features implementation |

## Structure

Each session log should include:

```markdown
# Session: YYYY-MM-DD - Topic

## Summary
Brief overview of what was accomplished.

## Changes Made
- File changes
- New features
- Bug fixes

## Decisions
Key architectural or implementation decisions.

## Learnings
Insights that should inform future work.

## Next Steps
What remains to be done.
```

## Usage

Session logs help:
- Track progress over time
- Document decision rationale
- Onboard new contributors
- Debug issues by understanding history
