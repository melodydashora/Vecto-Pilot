**Pattern:** CLAUDE.md gets written as *aspirational truth* (what the system *should* look like) and rots when the implementation diverges in either direction. Rule 12's contested-fact amendment (commit `0002b20c`) addresses "which source wins when docs disagree" but does not address "how does drift happen in the first place."

### Proposed Rule 15 (next session to draft and propose)

> **Rule 15: CLAUDE.md Reference Integrity.** When adding a new command, path, directory, or binary reference to CLAUDE.md, the reference MUST be accompanied by a path pointer to the file/script that implements it. Commands that do not resolve to an existing file at commit time are rejected by review. Removed-but-still-documented items must be removed from CLAUDE.md in the same commit that removes the implementation.

This would catch both drift directions:
- **Absent-but-documented** (the `bin/vecto-runner` case) — the linked path wouldn't resolve at commit time, caught by reviewer or pre-commit hook.
- **Present-but-mis-described** (the prod-DB case) — Rule 15 doesn't directly catch this; Rule 12's amendment handles it. Complementary coverage.
